================================================================================
                            BRIEF
================================================================================
The 3D SEO Website Deconstruction project is an innovative web application that 
enables businesses to visualize their website structure and SEO performance 
metrics in an interactive 3D environment. Rather than viewing SEO data in 
traditional tables or charts, this platform generates a real-time 3D 
representation of website architecture where:

• Each webpage becomes a 3D node/building in a virtual space
• SEO performance scores determine the height/size of each structure
• Link relationships create visual connections between pages
• Real-time metrics (load time, score, keywords) are displayed interactively
• Performance issues are highlighted with visual indicators (colors, glows)

Key Features:
1. Real-time 3D Website Crawling: Automatically crawls up to 15 pages deep
2. SEO Metrics Analysis: Extracts title, meta descriptions, headings, word count
3. Interactive 3D Visualization: Uses Three.js for WebGL rendering
4. Performance Scoring: Calculates SEO scores (0-100) per page
5. Link Mapping: Visualizes internal link structure and hierarchy
6. Issue Detection: Identifies SEO problems (missing titles, meta, etc.)
7. PDF Reports: Generates downloadable SEO audit reports
8. Real-time Updates: Live dashboard with analytics and recommendations

SECTION 1: 3D DATA STRUCTURE API (Backend - Flask)
================================================================================
Location: /api/seo/3d-structure
Method: GET
Parameters: url (website URL to analyze)

@app.route('/api/seo/3d-structure', methods=['GET'])
def get_3d_seo_structure():
    """Get 3D visualization data for a website"""
    url = request.args.get('url')
    
    if not url:
        return jsonify({"error": "URL parameter is required"}), 400
    
    # Validate URL format
    if not (url.startswith('http://') or url.startswith('https://')):
        url = 'https://' + url
    
    try:
        # Initialize crawler with conservative settings
        crawler = SEOCrawler(max_depth=2, max_pages=15, delay=1.0)
        
        # Crawl the website
        result = crawler.crawl_website(url)
        
        if not result['pages']:
            return jsonify({"error": "No pages found or unable to crawl the website"}), 404
        
        return jsonify(result), 200
        
    except Exception as e:
        logger.error(f"Error crawling website {url}: {str(e)}")
        return jsonify({"error": f"Failed to crawl website: {str(e)}"}), 500


SECTION 2: SEO CRAWLER ENGINE (Backend - Python)
================================================================================
Location: /backend/seo_crawler.py

Class: SEOCrawler
Key Methods:
- __init__(max_depth, max_pages, delay) - Initialize crawler
- can_crawl(url, base_domain) - Check robots.txt and domain validity
- extract_links(soup, base_url, base_domain) - Extract internal links
- extract_meta_data(soup, url) - Extract SEO metadata
  * Extracts: title, meta description, meta robots, canonical links
  * Analyzes: heading structure (H1-H6), word count, keywords
  * Detects: SEO issues and problems
- calculate_seo_score(page_data) - Calculate page score (0-100)
  * Scoring factors:
    - Title presence and length optimization
    - Meta description presence and length
    - Heading hierarchy (H1-H3)
    - Content word count
    - Canonical tags and robots meta tags
- crawl_page(url, depth, base_domain) - Crawl single page
- crawl_website(start_url) - Main BFS crawling algorithm

SECTION 4: SEO ANALYSIS ENGINE
================================================================================
The analyzer performs comprehensive SEO audits:

1. On-Page SEO Checks:
   - Title tag (optimal: 30-60 chars)
   - Meta description (optimal: 120-160 chars)
   - H1 tag (exactly 1 per page)
   - H2/H3 hierarchy
   - Image alt text
   - Internal linking density

2. Technical SEO Checks:
   - Robots.txt analysis
   - Sitemap detection
   - Canonical tags
   - Robots meta tags
   - Schema markup detection

3. Content Analysis:
   - Word count (minimum 300 words)
   - Keyword density
   - Readability score
   - Content freshness

4. Performance Metrics:
   - Page load time
   - Core Web Vitals (LCP, FID, CLS)
   - Mobile responsiveness
   - CSS/JS file sizes

Scoring Algorithm:
```
Base Score = 100
- 20 points for missing title tag
- 15 points for missing meta description
- 15 points for missing H1
- 10 points for low word count
+ 5 points for canonical tag
+ 5 points for robots meta tag
Final Score = Clamped between 0-100
```
